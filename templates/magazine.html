THE ORDER OF OPERATIONS

An operation is anything that uses an operator.
We use parentheses in a programming language to control the order of operations. Parentheses are always executed first.

Multiplication and division have a higher order than addition and subtraction, which means that they’re performed first.
In other words, if you enter an equation in Python, multiplication or division is performed before addition or subtraction

Don't bother with exponent for now. Operations are always evaluated from left to right when having the same precedence.

EXPRESSIONS AND STATEMENTS

Python programming has concepts similar to phrases and sentences, which are called expressions and statements.
You can combine values, variables, and operators to create small pieces of code called expressions, like "3 + 4"
Statements are single lines or short blocks of code that do something in a program, such as" diamonds = 4 + 12"

VARIABLES

Data is any information you might want to record, such as numbers, names or any kind of text — in Python, texts are called strings — , lists of items, and so on.

The variable’s name is always on the left of the equal sign, and the value you want to store is always on the right
eg : age = 22

Syntax Rules for Variables
You need to know a few syntax rules for naming variables so Python can understand them:

  • Don’t include symbols in your variable names, except for underscores (_), or you’ll get a syntax error. White space inside the variable name is not allowed also.
    • >
  • Don’t start a variable name with a number, as in 3player. Using numbers elsewhere in a variable name is fine, as in player3.
    • >
  • You don’t need to add spaces on either side of the equal sign: your program will run fine without them. But they do make the code easier to read, so it’s a good idea to add them.
    For example player='Little Vin' is equal to player = 'Little Vin' , adding space just makes them more readable.

COMMENTS

In any programming language a comment is a programmer-readable.
They are added to make the source code easier for humans to understand, and are generally ignored by compilers and interpreters.
In Python code, anything after a # symbol in a statement is a comment and is IGNORED by the Python Interpreter — think of them as code the machine don't see, as the robot in the figure above.
Reasons to use:

1.Comments can be used to explain Python code.

PYTHON NUMBERS

There are three numeric types in Python:

1.int

2.Float

3.complex


We will ignore complex numbers by now.

Also, don't worry if you don't know things like math with negative numbers or math with decimal points, Python will do the math for us.
Integers (int)

Integers are positive or negative whole numbers.
Values such as 10, 32, −6, 194689, and −5 are integers, but 3.14 and 6.025 are not.
Python also provides a built-in function called int() that converts any passed argument to an integer.

Floats (float)

Not all numbers are whole numbers.
Decimal points are used to represent values that can’t be described with whole numbers.
For example, you might have half (0.5) of an orange.
Numbers that use decimal points are called floating-point numbers, or floats.
Python also provides a built-in function called float() that converts any passed argument to a float number
